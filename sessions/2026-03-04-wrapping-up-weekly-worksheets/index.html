Wrapping up the weekly worksheets module

Ben Miller

March 4, 2026

We’re closing up this part of the project so that we can start focusing more on the backend model and web UI. So far what we have is a nicely formatted worksheet which is entirely defined by a special YAML format.

The instance data using this YAML is defined in the worksheets repo, but is loaded and transformed by the Elixir code in the sheetwork repo.

Ultimately, we have a canonical daily routine defined in weekly.yml. This includes every item that might go into a typical day’s routine, in order. To this list, we apply two core concepts:

  1. Tag filtering — items can be configured to only show or be excluded based on tags.
  2. Insertions — it’s possible to define lists specific to a given day, inserted at a certain point in the overall checklist.

Additionally, we have hint blocks, which in practice are used to define suggestions for what can be done with empty time after a given routine is finished and before the next one begins.

In theory the model is not complex, but in practice it’s easy to let it become that way anyway. What we have here is the result of probably hundreds of tiny changes, tweaks, and design decisions. It’s still in pretty rough shape but we’re trying to slow things down and get it into a nice, stable form for the time being.

Major refactors will be coming up, but I want to tie a bow on this so that I don’t have to think about it as I focus on other things such as the backend model and web application.

Remaining issues, or things I want to do before calling this done:

Low-hanging fruit:

  • YAML linter. The formatting is inconsistent, and that bothers me.
  • Consistent use of - vs _.
  • Naming — the day_tags header probably doesn’t make sense anymore.

Things that might require more thought:

  • Making use of insertions where possible. A number of lists in weekly.yml could be insertions but aren’t — cases where the same tag is applied to several adjacent items. Should be easy to refactor, but may involve adding a layer so that the insertion applies to the week instead of the day as it currently works.
  • Simplify the Typst interface. In particular, the YAML format that render_weekly.typ expects. For example, I don’t think the convenience syntax of --- asdf --\\ makes sense to have anymore. Cleaning this up is probably an entire dev session in its own right. For now, I just want to get a clearer picture of what it actually looks like.
